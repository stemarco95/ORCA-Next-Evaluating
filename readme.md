# An Architectural Approach to Monitoring Modular Reinforcement Learning Systems

This repository contains the code and scripts used to generate the results for the master thesis *"An Architectural Approach to Monitoring Modular Reinforcement Learning Systems"*.

---

## ⚙️ Installation

### Prerequisites
Before you begin, ensure you have [Miniconda](https://docs.conda.io/en/latest/miniconda.html) (or Anaconda) and Git installed on your system.

### 1. Clone this repository
```bash
git clone git@github.com:stemarco95/ORCA-Next-Evaluating.git
cd ORCA-Next-Evaluating

```

### 2. Set up the Conda environment

Create the virtual environment using the provided requirements file and activate it:

```bash
conda env create --file environment.yml
conda activate orca-next

```

### 3. Install the custom Highway-Env

This thesis relies on a customized version of the `highway-env` package. Clone it into a directory alongside your thesis repository and install it into the active conda environment in editable mode:

```bash
cd ..
git clone git@github.com:Heberleo/HighwayEnv.git
cd HighwayEnv
conda activate orca-next
pip install -e .

```

## 🚀 Usage Pipeline

The workflow is divided into three main stages: Pretraining, Experiments, and Evaluations.

### Stage 1: Pretraining

The pretrained models used in the thesis are already provided in the `pretraining/experiments/` directory. If you wish to retrain the agents, you can do so by following the instructions below.

To pretrain the agents, run the training script. The script will automatically store the best agent for each setting at `pretraining/experiments/<category>/<value>/best_model.zip`.

```bash
# From the root of the repository:
cd pretraining
python train.py

```

*Note: The fine-tuned models are not used in the current experiments, they serve as a feasability check if the generalized agent can be effectively fine-tuned to specific target distances.*

**Hyperparameters used for training:**
| Hyperparameter | Value |
| :--- | :--- |
| Total Timesteps | `20000` |
| Network Architecture (Actor & Critic) | `32 x 32` |
| Learning Rate | `1e-3` |
| Batch Size | `256` |
| Replay Buffer Size | `2000` |
| Discount Factor (γ) | `0.99` |
| Target Update Rate (τ) | `0.005` |
| Action Noise | `N(0, 0.1)` |
| Policy Delay | `2` |
| Seeds | `42, 43, 44` |

### Stage 2: Experiments

The model library is configured by default to point to the best models generated during the pretraining step. System configurations, shared monitoring configurations, and shared disturbance schedules for each scenario are located in `orca_next/configs/ScenarioX/`.

The main script is prepared to execute all three scenarios sequentially.

```bash
# From the root of the repository:
cd orca_next
python -m main

```

**Outputs:**
The results will be stored in the `orca_next/monitoring/` directory:

* `*.csv`: Raw execution traces logged by the Supervisor.
* `*.json`: Extracted operational phases.
* `*.png`: Plots of the operational phases and utility traces of the single systems.

### Stage 3: Evaluations & Figures

*Note: The thesis results are already provided in the respective `evaluation/scenario-X/` folders. If you ran new experiments, ensure you edit the paths in the evaluation scripts to point to your new results.*

**Important:** To allow the scripts to correctly import the `orca_next` package, all evaluation and plotting scripts **must be executed as modules from the root of the repository**.

**1. Generate Evaluation Heatmaps**
These scripts perform the parameter calibration across different nominal recovery times ($t_{nom}$) and utility thresholds ($U_{nom}$), generating the heatmaps compared in the thesis.

```bash
python -m evaluation.scenario-1.eval
python -m evaluation.scenario-2.eval
python -m evaluation.scenario-3.eval

```

**2. Generate Utility Traces**
To generate the combined utility traces for each scenario:

```bash
python -m figures.utility_traces

```

**3. Generate Additional Figures**
Other figures presented in the thesis can be generated by running their respective scripts located in the `figures/` directory (e.g., `python -m figures.<script_name>`).
